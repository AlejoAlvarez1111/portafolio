## Documentación

### Propósito y ventajas de la estructura usada en el ejercicio

En el ejercicio grupal se utilizó una estructura HTML semántica compuesta por etiquetas como `header`, `nav`, `main`, `section`, `article` y `footer`. El propósito de esta estructura es organizar el contenido del sitio web de forma clara, lógica y fácil de entender tanto para los desarrolladores como para el navegador.

Entre las principales ventajas se encuentran:

- Permite mantener el código ordenado y legible.
- Facilita la navegación dentro del proyecto y el mantenimiento del código.
- Mejora la accesibilidad, ya que las herramientas de lectura pueden interpretar mejor la estructura.
- Favorece el posicionamiento en buscadores (SEO).
- Permite escalar el proyecto agregando nuevas secciones sin afectar la organización existente.
- Facilita el trabajo colaborativo, ya que cada integrante puede identificar fácilmente cada parte del sitio.

### Importancia de la separación entre cliente y servidor en aplicaciones web

En las aplicaciones web es fundamental separar el cliente y el servidor. El cliente corresponde a la parte visible que se ejecuta en el navegador (HTML, CSS y JavaScript), mientras que el servidor se encarga de la lógica del sistema, el procesamiento de datos y la comunicación con bases de datos.

Esta separación es importante porque:

- Mejora la seguridad, ya que la información sensible se mantiene en el servidor.
- Permite que la interfaz y la lógica del sistema se desarrollen de forma independiente.
- Facilita el mantenimiento y la escalabilidad del proyecto.
- Permite reutilizar el servidor para diferentes clientes (web, móvil, etc.).
- Optimiza el rendimiento al distribuir responsabilidades entre cliente y servidor.
